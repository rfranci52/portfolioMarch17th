I followed the 4 hour tutorial and micro-analyzed, tried to follow it for about 2 weeks in my spare time. It was all I thought about during that period of time. I was Introduced to new languages like PHP, HTML and CSS.

The bootcamp that I am attending is Columbia Fullstack Coding Bootcamp by Trilogy. The first few week we learn to fundamentals of HTML and CSS and I finally get an understanding of what that ‘styles.css’ file does. The next week we focused on different approaches of how to build a front end website. We used media queries to understand what was actually going on. We also used Bootstrap to show how you can quickly build a powerful front end.
We were soon introduced to javascript and jquery. We started making games like a word guessing game and rock paper scissors just to get a feel of how the logic of javascript works. We eventually got into using APIs with javascript. We used the Giphy and OMDB APIs to get a basic understanding of how APIs work and what JSON is. That was the first third of the class and it ended with a group project. We were given teammates that were picked by the Teacher Assistants. I felt that is 2 week period was so far one of the most crucial. It tests your character and gives you an Mock project where you get a feel for how things would go in the real world. It adds an important dimension of coding that is a game changer. The collaborative dimension explores a group of people working symbiotically to achieve a common goal. My group consisted of 5 people and the first week of the 2 week project we had very little. We would pitch ideas and some were very ambitious but the problem was that we weren't really on the same page. There was some competing personalities and I did my best to mediate and reiterate the timeline and the importance of a Minimum Viable Product or MVP for short. The second week we had a MVP which was a googleMaps API which rendered a map. We connected that the the NYC Open Data API. The NYC Open Data website is very robust but the level of organization varies from dataset to dataset. Primarily in this group I was tasked with understanding how to produce the JSON values and once they were mapped on the googleMaps API, I was tasked with making them produce some unique information. This was challenging but at the same time very fulfilling. I felt that I was an important productive teammate and also a respectful teammate. I feel that both are equally important for collaboration.
Currently we are being introduced to Node js. We started off with the basics of using javascript in as the backend with node. I built an application that uses Spotify, BandsinTown and OMDB to search for (artists and songs), (concerts and showtimes) , and (movies and showtimes). This introduction was exiting because it comes full circle a bit with how I started with python and this node js is reminiscent of that. I was pleasantly surprised to find out that node has a mySQL library and we had an assignment that allowed me to use my knowledge of mySQL and stack that on top of my knowledge of javascript. I built a relational database Using mySQL and node js that from the terminal takes commands that are connected to query string parameters. This time creating a mySQL database was a piece of cake.
We were just introduced to express and a already see the power of it and this whole fullstack thing is really starting to make sence. Very soon I will Understand the MERN stack.
